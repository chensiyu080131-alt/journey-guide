## 一、产品概述

### 1.1 一句话定位
**跟着书本去旅行** ——  AI驱动的文学旅行攻略平台，让每座小城的故事从书页走进现实。

### 1.2 核心问题
中国有2844个县级行政区，其中90%以上的县城景点在小红书、马蜂窝等平台上搜不到几条有效攻略。旅游信息极度集中于热门城市，小城市的文化底蕴和独特体验被埋没。而每一座小城都有文学作品、历史人物、非遗传承——这些文化资产是现成的内容金矿，却没有被转化为可消费的旅行体验。

### 1.3 解决方案
用AI将文学作品、历史人物、非遗传承与真实地点关联，生成"文学旅行攻略"——用户选一本书/一个人物/一段故事，AI生成对应的实地旅行路线，覆盖书中场景还原、在地美食、历史文化、非遗体验等维度。

### 1.4 差异化
| 维度 | 传统平台（小红书/马蜂窝） | 寻城 |
|------|--------------------------|------|
| 内容入口 | 搜目的地 | 搜书/搜人/搜故事 |
| 覆盖范围 | 热门城市为主 | 全国2844个县城 |
| 内容来源 | UGC，依赖用户发布 | 文学作品+AI生成+公开数据聚合 |
| 信息密度 | 小城几乎空白 | 每城完整攻略，有叙事线 |
| 使用门槛 | 需要筛选大量帖子 | 一键出完整方案 |
| 个性化 | 弱 | 基于预算/时间/兴趣定制 |

## 二、目标用户

### 2.1 核心用户画像
- **自驾/自由行旅客**：想去小众目的地但找不到攻略
- **返乡探亲群体**：回老家想带家人周边玩，不知道附近有什么
- **文化旅行者**：对地方历史、民俗有好奇心
- **县域本地人**：想重新认识自己的家乡

### 2.2 用户场景
> "周末从杭州开车去丽水松阳，搜了小红书只有3篇游记，不知道怎么安排一天行程，也不知道哪碗面是本地人吃的。"

> "过年回老家县城，想带爸妈出去转转，但百度搜出来的都是过时信息。"

## 三、功能设计

### 3.1 核心功能（MVP / 黑客松Demo）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 📚 书籍/人物入口 | 选择一本书/历史人物/故事，生成对应实地路线 | P0 |
| 🗺️ 智能路线 | 输入目的地+天数，自动生成日程路线 | P0 |
| 🍜 在地美食 | 非网红店、本地人吃的、时令推荐 | P0 |
| 📖 历史科普 | 每个景点的文化故事、历史脉络，关联文学原文 | P0 |
| 💰 预算匹配 | 输入预算自动调整消费档位 | P1 |
| ⏰ 时间感知 | 根据当前时间推荐"现在还能去哪" | P1 |
| 🎭 在地体验 | 本地人才知道的体验（赶集日、时令水果、拍照点） | P1 |
| 🗣️ 方言速查 | 当地方言常用语+文化禁忌 | P2 |

### 3.2 亮点功能（路演加分项）
- **AR景点识别**：拍建筑/碑文弹出历史故事，AR场景还原（参考常熟研学方案：扫实景触发历史3D动画重现）
- **非遗专区**：每个县城的非遗项目（手作/曲艺/民俗）做独立卡片，含体验预约入口
- **亲子双角色**：孩子做"文化侦探"解谜探索，家长做"后勤管家"安排食宿，双线并行
- **研学模式**：按年龄分层任务（小学生→探秘答题，中学生→模拟操作，大学生→深度调研）
- **四季时令**：攻略随季节变化，春天采茶、夏天赶集、秋天收割、冬天温泉，不同时间来看到不同内容
- **"我的家乡"模式**：本地人视角写攻略，反哺平台

### 3.3 交互流程（Demo路径）
```
打开页面 → 选择一本书/人物/故事（如《沙家浜》）或输入目的地（如"常熟"）
    ↓
AI生成文学旅行攻略：书中场景→真实地点映射+路线+美食+历史+非遗
    ↓
点击具体场景卡片 → 展开详情（原文片段+地图定位+文化故事+实地照片）
    ↓
选择偏好（预算/天数/兴趣）→ 生成定制路线
    ↓
"出发"按钮 → 进入行程模式（时间感知推送下一个推荐）
```

## 四、技术方案

### 4.1 技术栈
| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Next.js 14 + TypeScript + Tailwind CSS | 快速开发，SSR利于SEO |
| UI | shadcn/ui | 黑客松首选，开箱即用 |
| 地图 | 高德地图JS API | 国内最全POI数据 |
| AI | 通义千问API / DeepSeek API | 国产LLM，中文理解强 |
| 数据 | 公开旅游数据+政府文旅公开信息 | 不依赖爬虫 |
| 部署 | Vercel | 零配置部署 |

### 4.2 AI架构
```
用户输入（目的地+偏好）
    ↓
Step 1: POI检索 → 高德API获取景点/餐饮/住宿POI
    ↓
Step 2: 信息聚合 → LLM整合公开数据生成结构化攻略
    ↓
Step 3: 路线优化 → 按地理位置+开放时间+用户偏好排序
    ↓
Step 4: 个性化调整 → 根据预算/时间/兴趣标签微调
    ↓
输出：结构化攻略JSON → 前端渲染
```

### 4.3 数据策略
- **景点POI**：高德地图API（免费额度足够Demo）
- **历史文化**：LLM知识 + Wikipedia/百度百科结构化
- **美食数据**：高德餐饮POI + 大众点评公开信息
- **在地体验**：LLM基于地域知识生成 + 用户反馈校验

## 六、市场分析

### 6.1 市场规模
- 2024年百强县旅游总收入**1.9万亿元**，占全国近三成
- 县域旅游接待量占全国超60%，收入贡献超40%
- 2026年五一县域预订暴涨**128%**，中低线城市异地用户环比增速205%
- 年增速12-15%，总量超2万亿的真实增长市场

### 6.2 竞品分析
| 平台 | 县城覆盖 | 核心短板 |
|------|---------|---------|
| 小红书 | 有渗透但浅层种草 | 旅游转化率仅4.7%，无决策闭环 |
| 马蜂窝 | 攻略质量高但覆盖有限 | 整体认知度35%，县城内容有限 |
| 携程 | 协助174个县级搭建官网 | 偏预订闭环，攻略非核心 |
| 抖音 | 播放量4025亿次 | 信息密度低，无精度 |
| AI旅游产品 | 聚焦大城市/B端 | 无一专注中国县城攻略 |

**关键结论**：没有任何平台系统性解决"小城怎么玩"的信息需求，本项目在竞品格局中无直接对手。

### 6.3 政策利好
- **《旅游强国建设"十五五"规划》**（2026.7.8发布）："发展县域旅游"+"发展智慧旅游"并列提出
- **2026中央一号文件**："发展'小而美'文旅业态"
- **《人工智能+行动意见》**：2030年智能终端普及率超90%
- **数字中国创新大赛**：专设"数字文旅赛道"

### 6.4 AI+旅游赛道趋势
- 40%旅行者已使用AI规划行程，96%会再次使用
- 玩点旅行Pre-A超1亿元，AI收入同比增长10倍
- "AI游利川"已验证：停留时长+30%，二次消费+45%，投诉率-32%

### 6.5 技术可行性
- **LLM幻觉问题在县城更严重**（训练数据不足），RAG+本地知识图谱是必要条件
- **AR景点识别**已达商用水平（mAP@50=0.995），但县城3D建模基础薄弱需分阶段投入
- **核心风险**：数据获取成本、商业化路径、用户信任建立周期

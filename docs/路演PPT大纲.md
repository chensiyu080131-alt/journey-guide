## 第 4 页：产品是什么（v0.2 文化地理引擎）

**一句话定位**：
> 选一本书 / 一个人 / 一座城 / 一种文化载体 → AI 生成**可落地、有原文对照**的文学旅行路线

**产品架构（一页图）**：

```
文化载体（书·影·游·音·动）
        ↓
   城市 / 书单 / 人物
        ↓
  原文提取 → POI 验证 → 行程编排 → 地图探索
```

**三大 C 端入口**：
| 入口 | 用户动作 |
|------|----------|
| 📖 预设路线 | 首页滑动封面 → 一键进入常熟/江苏经典线 |
| 📖 **跟书旅行** | 任意页面浮窗 → 输入书名 → AI 生成专属攻略 |
| 📍 搜一座城 | 城市 + 天数 + 兴趣 → AI 即时生成 |

**B 端入口**：`/dashboard` 文旅局工作台（白皮书 + 内容资产看板）

## 第 11 页：技术方案（v0.2 已实现）

```
用户输入（书名/城市/摘录）
    → LLM 提取书中地点 + 原文引文
    → 高德 POI 验证（地址/坐标）
    → LLM 基于锁定地点表编排行程
    → Map Explorer 渲染 + 沿路导航
```

| 层 | 选型 |
|----|------|
| 前端 | Next.js 14 + TypeScript + Tailwind + AI SDK |
| 服务端 | API Routes（`/api/book-guide` `/api/chat` `/api/generate-guide`） |
| AI | OpenAI 兼容 API（密钥仅服务端，不进浏览器） |
| 地图 | 高德 JS API + REST POI 搜索 |
| 部署 | **standalone + ECS + pm2 + nginx**（已配置 CI） |

**抗幻觉三板斧**：
1. **摘录优先** — 用户粘贴原文，引文从书中来  
2. **POI 验证** — 高德搜索，标注「已验证 / 未验证」  
3. **地点表锁定** — 行程只能使用已提取地点，禁止 AI 凭空加网红店

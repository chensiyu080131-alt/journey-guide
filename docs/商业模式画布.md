# 寻迹商业模式画布

> 项目定位：AI 驱动的文化旅行攻略平台，把书籍、城市、游戏、音乐等文化载体里的线索，转化为现实中可行走、可传播、可运营的路线。

## 1. 目标用户

### C 端用户

- 文学旅行、城市漫游、研学旅行用户
- 周末短途游用户，尤其是江浙沪县域与小城目的地用户
- 喜欢从书籍、音乐、游戏、影视 IP 进入现实地点的年轻用户
- 需要低门槛路线规划的亲子家庭和学生群体

### B 端客户

- 文旅局、景区管委会、城市运营公司
- 研学机构、旅行社、地方文化馆/博物馆
- 需要把地方文化资源转成路线、专题和传播素材的城市品牌方

## 2. 核心价值主张

- **从“去哪儿”升级为“为什么去”**：传统攻略解决交通、餐饮、门票，寻迹解决文化线索和城市叙事。
- **小城文化资产结构化**：把地方志、文学作品、人物故居、非遗、美食、景区点位整理成可复用数据。
- **AI 降低内容生产成本**：通过迹录员、跟书旅行、白皮书生成器，快速生成路线、讲解词、活动方案和点位卡。
- **C 端体验与 B 端运营共用一套底层数据**：同一份点位数据可支撑地图路线、互动任务、白皮书、研学产品和传播内容。

## 3. 产品形态

| 模块 | 面向对象 | 当前能力 |
| --- | --- | --- |
| 首页文化载体入口 | C 端 | 从书籍、城市、游戏、音乐进入路线 |
| 跟书旅行 | C 端 | 输入书名/作者/城市，生成文学旅行路线 |
| 迹录员智能体 | C 端 | 全局浮窗对话，按书名、城市、偏好生成攻略 |
| 地图路线探索 | C 端 | 点位地图、原文对照、实景说明、路线卡片 |
| 文旅局工作台 | B 端 | 城市文学白皮书、内容资产看板、合作表单 |
| 互动任务/探险模式 | C 端/B 端 | 诗词诵读、知识问答、古籍寻宝、书法临摹 |

## 4. 收入模式

### C 端收入

- 高级路线包：主题路线、研学路线、周末短途路线
- AI 个性化规划：多天行程、预算优化、同行人偏好定制
- 会员订阅：收藏路线、离线攻略、深度讲解、专属点位

### B 端收入

- 城市文化资产梳理服务
- 文旅局/景区 SaaS 工作台
- 白皮书、专题路线、活动策划共创
- 城市 IP 内容包：讲解词、图文稿、短视频脚本、研学手册

## 5. 渠道策略

- 黑客松/路演展示：以常熟 Demo 和《人间滋味》路线证明产品可用性
- 小红书/B站/视频号内容种草：用“跟一本书去旅行”的内容吸引 C 端
- 文旅局与研学机构试点：从常熟、扬州、高邮等有明确文学资产的城市切入
- GitHub 与产品文档：展示工程完整度、数据结构和商业思考

## 6. 成本结构

- AI 调用成本：路线生成、白皮书、对话式规划
- 地图与 POI 成本：高德地图、地点检索、路线规划
- 内容校验成本：原文、地点、史实、文旅资料核验
- 产品研发成本：前端体验、数据结构、B 端工作台、部署运维

## 7. 关键资源

- `src/data/changshu-spots.json`：常熟 4 条路线、26 个结构化点位
- 文学/城市/人物路线数据库
- 迹录员智能体与 AI 生成链路
- 文旅局白皮书模板和内容资产看板
- 路演 Demo 地址：<http://47.109.91.112:8080/>（黑客松期间可用）

## 8. 关键指标

### 产品指标

- 路线生成成功率
- 点位具备原文/出处/实景对照的比例
- 用户从首页到路线页的转化率
- 迹录员对话后生成攻略的转化率

### 商业指标

- 城市资产梳理成本
- 单城市可产出路线数
- B 端白皮书生成与人工校对耗时
- 路线包转化率与复购率

## 9. 阶段规划

| 阶段 | 目标 | 重点 |
| --- | --- | --- |
| P0 Demo | 证明“文化线索 -> 地图路线”成立 | 首页、跟书旅行、常熟路线、迹录员 |
| P1 文旅局工作台 | 证明 B 端内容生产价值 | 白皮书、内容资产看板、城市路线包 |
| P2 探险模式 | 提升 C 端停留和传播 | 互动任务、打卡、知识问答、研学玩法 |
| P3 多文化载体 | 扩展到游戏、音乐、影视 | 统一抽取人物、地点、情节、城市关系 |

## 10. 一句话商业判断

寻迹的机会不在于再做一个通用攻略平台，而在于把“分散的地方文化资源”产品化、结构化、路线化，帮助小城以更低成本生产可传播、可体验、可变现的文化旅行内容。

## 四、跟书旅行 AI 向导（v0.2 核心新功能）

全局浮窗向导，三步收集信息 → 调用大模型 → 结构化攻略预览 → 跳转完整页面。

### 4.1 用户流程

```
任意页面点击「📖 跟书旅行」
  Step 1：书名（必填）+ 作者
  Step 2：目标城市（可选）+ 天数 + 兴趣 + 预算 + 偏好
  Step 3：书籍摘录（强烈推荐）
  → POST /api/book-guide
  → 浮窗预览 →「查看完整攻略」→ /guide/generated/
```

### 4.2 服务端生成管线（两阶段锁定）

| 阶段 | 动作 | 输出 |
|---|---|---|
| **① 城市推断** | 用户填写 / 书名规则 / LLM 推断 | 主线城市 |
| **② 地点提取** | LLM 从书中提取 ≥6 个地点 + **真实原文引文** | `location对照` |
| **③ POI 验证** | 高德 REST 文本搜索验证书中地点 | 地址、坐标、开放时间 |
| **④ 行程编排** | LLM 仅基于已锁定地点表排日程 | `dayPlans` + 交通 + 餐饮 |
| **⑤ 原文回填** | 强制将书中原文写入每个行程点位 | `originalText` / `realityNote` |

### 4.3 结构化输出

| 字段 | 说明 |
|---|---|
| `location对照` | 书中称呼 ↔ 现实 POI，含原文、出处、验证状态 |
| `dayPlans` | 按天行程，每点含原文 + 实景对照 |
| `diningRecommendations` | 餐饮推荐 + 书中关联 |
| `dayTransports` | 每日交通摘要 |
| `meta` | 提取数、POI 验证数、含原文点位数、是否演示模式 |

### 4.4 使用要点

- **目标城市可选**：不确定书中涉及哪些城市时可留空，AI 根据书名/摘录推断
- **粘贴摘录效果最好**：无摘录时引文由 AI 根据书名推断，准确度较低
- **演示模式**：未配置 `LLM_API_KEY` 或 `USE_MOCK=true` 时返回常熟示例路线，浮窗顶部有黄色提示
- **需重启 dev**：修改 `.env.local` 后须重启 `npm run dev` 才生效

## 八、技术架构

```
用户入口
  ├─ 预设路线 → mock-data / city-guides
  ├─ 城市书单 → city-books.ts → /guide/[city]/books/
  ├─ 搜目的地 → /api/generate-guide → llm-server
  ├─ 跟书旅行 → /api/book-guide → book-guide-server
  │                ├─ LLM 提取地点 + 原文
  │                ├─ poi-search（高德 REST）
  │                └─ LLM 编排行程（锁定地点表）
  └─ AI 对话 → /api/chat → ai-personas

展示层
  GuideMapExplorer（地图优先）
    ├─ amap-loader / amap-routing / map-route-visual
    ├─ SpotCard / SpotDetailDrawer
    ├─ AIBookSpirit / LiteraryDetective
    └─ BookGuideFloat（全局浮窗）
```

## 十二、总结

寻城 v0.2 从「常熟 Demo 静态站」演进为**可服务端 AI 生成、可全国扩展的文学旅行平台**：

1. **原文对照** — 书中写的，就是脚下走的（核心差异化）
2. **跟书旅行** — 任意书名 + 摘录 → 结构化攻略 + 地图（v0.2 亮点）
3. **常熟 4 条 + 江苏多城** — 可落地、可路演、可对接文旅局
4. **AI 书灵 / 文学侦探** — 路线深度互动，提升停留与传播
5. **文旅局工作台** — B 端白皮书与资产看板，支撑商务洽谈
6. **地图沿路导航** — 黄虚线 + 详情标记，接近专业漫游地图体验

**适合场景：** 黑客松路演、常熟/江苏文旅局洽谈、跟书旅行产品验证、Vercel / ECS 部署。
